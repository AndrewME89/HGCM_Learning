Troubleshooting iHMS

This procedure details the steps to reboot the servers located in the Level 1 Administration offices. Regular server reboots are essential for maintaining system performance and stability.

This procedure applies to the weekly server reboot, typically performed on Sunday night shifts.

Procedure
  1. Close HMS Instances: Ensure all instances of HMS are closed on all front desk computers, including the Reservations computers. If performing the reboot during the day shift, inform all Front Office and Admin staff to close HMS.

  2. Locate Servers: Enter the server room. The relevant servers and interface screens are located on the left-hand side, housed on a wooden shelf.

  3. Wake Interface Screens: The interface screens will be in sleep mode. Press the number 1 button and move the mouse to wake the first screen. If it remains black, check its power connection and ensure it's turned on. If still unresponsive, press any number on the interface server screen and repeat. Repeat this process for screens 2 and 3.

  4. Identify Servers for Reboot: Reboot servers 1, 3, and 5. These are essential for HMS synchronization with email, phones, internet, and movies.

    • Server 1: melb@131

    • Server 3: melb@131

    • Server 5: chancellor

  5. Reboot Servers:

    • Click the corresponding number on the interface screen (1, 3, or 5).

    • Either use the Start menu to initiate a restart or press Ctrl + Alt + Del and select "Restart."

    • In the comment box, enter "WEEKLY REBOOT" as the reason for the restart.

    • Click "OK."

  6. Log Back In: After the reboot, log in to all servers using the provided passwords.

  7. Troubleshooting:

    • If a server displays a black screen, press "Enter" to start Windows normally or wait for the automatic timer.

    • Verify that all servers are logged in and necessary programs are loading and operational.

    • If issues persist with phones, email, internet, or movies, reboot server 5 again without closing HMS or unplugging the external hard drive.

    • For any unresolved problems, contact Hendra upon his arrival for assistance.

Note:
  • Rebooting servers is a critical maintenance task.

  • Follow the steps carefully and seek assistance if needed.

  • Ensure all systems are operational before concluding the reboot process.
